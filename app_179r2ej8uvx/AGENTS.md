# UI 设计指南

> **设计类型**: App 设计（应用架构设计）
> **确认检查**: 本指南适用于可交互的应用/网站/工具。

> ℹ️ Section 1 为设计意图与决策上下文。Code agent 实现时以 Section 2 及之后的具体参数为准。

## 1. Design Archetype (设计原型)

### 1.1 内容理解

- **目标用户**: 团队协作成员与管理员，移动端高频操作待办事项，桌面端兼顾管理后台
- **核心目的**: 高效完成任务流转、状态追踪与人员审批，建立清晰可控的工作秩序感
- **情绪基调**: 专注高效 / 清爽不疲劳；避免信息过载、视觉噪音、廉价模板感

### 1.2 设计方向

- **Design Style**: Rounded 圆润几何 — 移动端优先的待办工具需拇指友好触控区，pill 按钮 + 柔和阴影降低认知负荷
- **Application Type**: Mobile-first Team Task Manager — 决定卡片流布局与底部导航倾向
- **Aesthetic Direction**: 低饱和中性底色承载高密度任务信息，仅用左侧色条与状态标签传递语义，克制而专业

## 2. Color System (色彩系统)

**色彩关系**: 石板蓝主色 + 暖灰底色 + 深墨文字，状态色从主色色相偏移衍生
**配色设计理由**: 长时间高频使用需低视觉疲劳，石板蓝比纯蓝更沉稳专业，暖灰底避免冷白刺眼
**主色推导**: primary 取石板蓝用于「进入工作台」「标记完成」等核心行动，呼应团队工具的可信赖感
**使用比例**: 60% 背景/卡片中性色 / 30% 次要文字与边框 / 10% primary 收敛于主按钮、FAB、激活态Tab

### 2.1 主题颜色

| Token                | HSL 值              | 说明                                     |
| -------------------- | ------------------- | ---------------------------------------- |
| `background`         | hsl(220 14% 96%)    | 暖灰页面底色，降低长时间使用的视觉疲劳     |
| `card`               | hsl(0 0% 100%)      | 白色卡片容器，与底色形成轻层次             |
| `foreground`         | hsl(222 47% 11%)    | 深墨主文字，高可读性                       |
| `muted-foreground`   | hsl(220 9% 46%)     | 次要说明文字、时间戳、placeholder          |
| `primary`            | hsl(215 60% 48%)    | 石板蓝主交互色，用于主按钮与关键操作       |
| `primary-foreground` | hsl(0 0% 100%)      | 主按钮上的白色文字                         |
| `accent`             | hsl(215 40% 94%)    | 次级交互反馈背景（hover/focus/skeleton）   |
| `accent-foreground`  | hsl(215 60% 38%)    | accent 背景上的深色文字                    |
| `border`             | hsl(220 13% 91%)    | 卡片边框、分割线，低存在感                 |

### 2.2 导航区配色

- **基调关系**: 复用主配色系统，底部导航栏使用 `card` 背景 + 顶部细线 `border` 分隔
- **关键状态**: 激活态图标+文字使用 `primary`，非激活态使用 `muted-foreground`；激活态对比度 ≥ 4.5:1
- **边界与背景**: 非透明 `card` 背景，顶部 1px `border` 分隔内容区，确保滚动时层级清晰

### 2.3 语义颜色

| 用途       | HSL 值              | 衍生逻辑                                   |
| ---------- | ------------------- | ------------------------------------------ |
| 待办/进行中 | hsl(32 95% 55%)     | 橙色系，左侧竖条+状态标签，温暖但不过激     |
| 已完成     | hsl(152 60% 42%)    | 绿色系，表示任务闭环，饱和度适中不刺眼       |
| 删除/拒绝  | hsl(0 72% 51%)      | 红色系，仅用于删除按钮与拒绝操作，需二次确认 |
| 待审核     | hsl(45 93% 55%)     | 黄色系，人员管理页审核面板底色，醒目区分     |

## 3. Typography (字体排版)

- **Heading**: Inter, system-ui, -apple-system, sans-serif
- **Body**: Inter, system-ui, -apple-system, sans-serif
- **字体策略**: Inter 提供优秀的数字与小字号渲染，适配任务列表密集信息；回退至系统字体确保移动端原生体感

## 4. Layout Strategy (布局策略)

- **导航意图**: 移动端优先的持久型底部导航（待办列表/已完成/人员管理/设置），桌面端自动切换为顶部Tab或侧边栏；至多一套全局导航，非透明背景
- **页面架构**: 移动端满屏垂直卡片流，桌面端内容居中 `max-w-[960px]` 双列网格；登录页全屏居中卡片
- **响应式**: 移动端单列卡片+吸顶筛选栏+FAB；桌面端双列网格+表格视图，详情页附图区自适应宽度

## 5. Visual Language (视觉语言)

- **形态参数**: 圆角 `rounded-xl (0.75rem)` · 阴影 `shadow-sm` · 间距基调 `standard (gap-4/p-5)`
- **识别签名**: 「卡片左侧4px色条标识状态」「负责人头像堆叠+数字角标」「FAB悬浮圆形按钮 rounded-full」
- **装饰策略**: 无额外装饰元素，仅通过色条、头像堆叠、状态标签建立视觉记忆点
- **动效原则**: 快速响应 150-200ms，状态切换/弹窗淡入/附图全屏查看流畅过渡
- **可及性**: 正文对比度 ≥ 4.5:1；附图全屏查看加 `bg-black/60` 遮罩；所有交互元素有明确 hover/focus 反馈

## 6. Component Principles (组件原则)

- **状态完整性**: Button/Input/Card/Tab 覆盖 Default/Hover/Active/Focus/Disabled；表单必填未填时 Input border 变 `destructive` + 下方提示文字
- **层级清晰**: Primary 按钮填充 `bg-primary`，Secondary/Ghost 按钮使用 `accent` 背景；删除按钮独立 `destructive` 样式并需二次确认弹窗
- **一致性**: 状态标签统一胶囊形 `rounded-full px-2.5 py-0.5 text-xs`；负责人标签统一头像+名称紧凑排列；颜色只用 Color System 语义角色

## 7. Image Direction (图片与视觉资产，按需)

- **Image Role**: 无强制图片需求，优先通过排版、色条、头像堆叠和局部图形建立视觉记忆点
- **Image Art Direction**: 无
- **Image Prompt Keywords**: 无
- **Image Avoidance**: 避免通用商务插画、科技渐变背景、无主题抽象图案；待办附图为用户自主上传，不提供预设占位图

## 8. 应避免 (Anti-patterns)

- ❌ 大面积高饱和色块或渐变背景：违背「长时间使用不疲劳」的核心诉求，干扰任务信息阅读
- ❌ 桌面端与移动端使用完全不同的导航体系：应保持同一套导航逻辑，仅调整形态（底部→顶部），避免维护两套交互心智
- ❌ 在卡片内堆砌过多装饰元素（图标、色块、徽章）：信息密度已通过色条+头像+时间表达，额外装饰增加认知负担
